Audit des profils

Outil local pour voir tous les documents profils, repérer les noms « test », les profils sans activité (aucune ligne famille_membres ni acces_ressource), et ceux que le script cleanup-orphan-profils.mjs pourrait supprimer (sans réservation liée). Cette page reste en lecture seule : la suppression se fait uniquement via scripts Admin SDK.

Les règles Firestore du projet exigent un utilisateur Firebase connecté (request.auth != null). Un simple code tapé dans la page ne peut pas remplacer ça côté Firestore. En pratique, votre « code secret » = le mot de passe d’un compte Firebase que vous créez uniquement pour l’admin (voir ci-dessous).

Ce navigateur a un code enregistré (famresa_diag_admin_code). Saisissez le même code que pour le diagnostic admin dans l’app, ou videz ce stockage.

Même origine que l’app, pas file:// — firebase-config.js à la racine.
Créer le compte admin (une fois)

Option A — script (recommandé) : avec la clé Admin du projet, à la racine du dépôt :

npm run audit:user:create -- --email=votre-audit@exemple.com
npm run audit:user:create -- --email=votre-audit@exemple.com --reset-link

Le script crée l’utilisateur sans mot de passe ; avec --reset-link, collez le lien affiché dans le navigateur pour choisir votre mot de passe. Sinon, définissez-le dans la console (Utilisateurs → ⋮).

Option B — console Firebase

  1. Authentication → activer E-mail / Mot de passe si besoin.
  2. Onglet Utilisateurs → Ajouter un utilisateur : e-mail + mot de passe que vous choisissez.
  3. Revenir ici et remplir les champs, puis Se connecter et charger.
